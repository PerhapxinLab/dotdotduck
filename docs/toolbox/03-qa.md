# qa

Multi-candidate question answering. **No automatic chunking** — your DB rows are
your units. Returns top-K candidates with a `locator` so the agent (or webagent)
decides how to use them.

## Why no chunking

- Auto-chunking always boundary-cuts in the wrong place.
- Web content already has DOM boundaries (heading, section, paragraph).
- DB content already has row boundaries.
- If you want chunks, you chunk yourself — store each chunk as a row.

The toolbox returns a `locator` per candidate:

| `locator.type` | What it means | What the agent does |
|----------------|--------------|---------------------|
| `static` | Inline answer | Display directly |
| `dom`    | `{ url, selector, anchor }` | Navigate to URL, scroll to selector |
| `row`    | `{ table, rowId }` | Fetch row from host DB, render |
| `live`   | Arbitrary payload from a dynamic source | Source's contract |
| `custom` | Host-defined | Host's renderer handles |

## Usage

```typescript
import { createQA, builtin } from '@perhapxin/dddk/toolbox/qa';

const qa = createQA({
  sources: [
    // FAQ: question(s) → answer
    builtin.sources.faq({
      rows: hostFAQs,
      adapter: (row) => ({
        id: row.id,
        question: row.q,
        variants: row.variant_qs,
        answer: row.a,
      }),
    }),
    // DOM: pointer to a page section
    builtin.sources.dom({
      rows: hostHelpPages,
      adapter: (row) => ({
        id: row.page + '::' + row.section,
        text: row.heading + ' ' + row.brief,
        url: row.url,
        selector: row.dom_selector,
        anchor: row.anchor,
      }),
    }),
    // Row: pointer to a DB row (e.g., product manual entry)
    builtin.sources.row({
      rows: hostManuals,
      adapter: (row) => ({
        id: row.manual_id,
        text: row.title + ' ' + row.summary,
        table: 'product_manuals',
        rowId: row.manual_id,
      }),
    }),
    // Live: dynamic source (e.g., inventory)
    {
      id: 'live_inventory',
      query: async (q) => {
        if (!/還有貨|stock/.test(q)) return [];
        const sku = extractSKU(q);
        return [{
          id: `inv-${sku}`,
          text: `stock for ${sku}`,
          locator: { type: 'live', payload: await fetchStock(sku) },
        }];
      },
    },
  ],
  topK: 5,
  scoreThreshold: 0.3,
  ambiguousGap: 0.15,
});

const candidates = await qa.ask('怎麼開機', {
  context: { customerId, currentPage: '/help', locale: 'zh-TW' },
});

if (qa.ambiguous(candidates)) {
  // Top results are too close — let agent / user pick
}
```

## Agent handoff (the canonical pattern)

```typescript
const candidates = await qa.ask(userQuestion, ctx);

await webagent.act({
  goal: 'Answer the user question',
  context: { question: userQuestion, candidates },
  capabilities: ['navigate', 'fetch_row', 'summarize', 'speak'],
});

// webagent reads the locators:
//   - 'static' → use answer directly
//   - 'dom'    → navigate(url) + scroll_to(selector)
//   - 'row'    → fetch_row(table, rowId) → reason
//   - 'live'   → use payload
```

## Confidence

Each candidate is labelled `high` / `medium` / `low` based on its score ratio to top.
Use `qa.ambiguous(candidates)` to detect when top-1 and top-2 are too close.

## Adding host-specific sources

```typescript
{
  id: 'internal_kb',
  query: async (q, ctx) => {
    const r = await fetch(`/internal/kb?q=${encodeURIComponent(q)}&customer=${ctx.customerId}`);
    const docs = await r.json();
    return docs.map((d) => ({
      id: d.id,
      text: d.title,
      locator: { type: 'custom', kind: 'kb_article', data: d },
      score: d.score,
    }));
  },
}
```

Source contract:
- `static()` returns candidates indexed once at registration (re-call `qa.rebuildStaticIndex()` to refresh)
- `query(q, ctx)` returns candidates per-query (no pre-indexing)
- Either or both
