Working tree state at session end. Nothing committed. Nothing pushed.

## Done — landed in working tree

### Foundation polish

- **1.3 SDK i18n stub** — voice module labels (`listening` /
  `unsupported` / `failed`) added to `sdk-i18n.ts`. `VoiceModule` now
  uses `sdkString(locale, ...)` for fallback when host doesn't supply
  overrides. New `VoiceModule.setLocale(locale)` for runtime locale
  switch.
- **1.5 disableBuiltinActions split** — new `excludeTools` and
  `disableAutoPauseAfterNarrate` config fields on
  `WebAgentConfig`. Old `disableBuiltinActions` kept as alias with a
  console deprecation warn.
- **1.6 tools.flush()** — new `ToolsRegistry.flush()` returns when any
  in-flight agent turn completes, so post-flush registrations land in
  the next turn deterministically.
- **2.3 Cross-origin + auth-required failure reasons** — new
  `'cross_origin'` and `'auth_required'` variants in
  `ActionFailureReason`.

### New tools

- **3.1 Generic action additions** — `escalate_to_human`, `track_intent`,
  `summarize_page` added to `builtinActions`. (`summarize_page` is a
  schema-only stub; runtime LLM hand-off lands as a follow-up.)
- **3.3 Adapter interfaces** — full TS interfaces for
  `EcommerceAdapter`, `FeatureIndex`, `ManualAdapter`,
  `CustomerProfile` in new `src/adapters/`. Re-exported from package
  root.

### Host-experience

- **4.7 escalate_to_human** — action + new `agent_escalated`
  intent-stream event type.

### Multi-model + Notes

- **6.2 Notes** — full `NotesAdapter` interface +
  `IndexedDBNotesAdapter` default backend +
  `createDefaultNotesAdapter()` factory. Lives in new `src/notes/`.

### Data ingest

- **7.1 Event schema** — `EVENT_SCHEMA_VERSION` (currently `1.0.0`)
  plus the full event category + props type catalog in
  `src/ingest/schema.ts`. Helper `makeEvent(...)` constructs
  well-formed events.
- **7.3 dddk.track() API** — new `DotDotDuck.track(name, props?)`
  method emits a `custom_track` intent event. The standard
  product-analytics surface engineers recognise.
- **7.4 Transport layer** — `Transport` class with batching, retry
  with backoff, ordering, beacon-on-unload, queue cap. Sinks shipped:
  `HttpSink`, `DebugSink`, `MultiSink`. Lives in
  `src/ingest/transport.ts`.

### Storage

- **11.1 + 11.2 Storage adapter** — full `StorageBackend` interface
  (event / document / vector / rollup / audit stores) in
  `src/storage/types.ts`. Default `createIndexedDBBackend()` +
  `createNoopBackend()` (SSR fallback) implementations in
  `src/storage/indexeddb.ts`. Audit store implements SHA-256 hash
  chain with `verify(orgId)` integrity check.

### Streaming agent envelope (1.9 — partial)

## Follow-ups (not done; deliberate skip in autonomous session)

- **1.9 per-character incremental streaming** — the parser bails when
  a single string value is split across `feed()` calls. Fixing this
  requires making `parseSingleAction` + `streamNarrateString`
  resumable (proper coroutine-style state). Architecture decision:
  the semantic-event API is the load-bearing part of 1.9; per-char
  resumption is an optimisation that can land later without breaking
  the API.
- **1.9 provider-side support** — `StreamChunk` needs a `toolArgsDelta`
  field; `OpenAIProvider` needs to forward incremental tool-call args.
  Once both land, wire `StreamingEnvelopeParser` into `loops.ts`
  behind `enableStreamingEnvelope`.
- **1.10 Multi-vendor LLM adapters** — needs real API key testing per
  provider. Skipped to avoid burning budget on a sleeping host's
  account.
- **1.1 Memory unification** — big refactor; skipped because it
  affects multiple host integrations and risks breaking demo.
- **Multi-step workflow items (2.1 / 2.2 / 2.4 / 2.5 / 2.6 / 2.7 /
  2.8)** — each is multi-day; left for focused implementation.
- **Vertical adapters (5.x)** — interfaces shipped (3.3); skill
  orchestration logic + surface templates left for follow-up.
- **Search engine (8.x)** — storage layer ready; search runtime
  worker lives in dddk-frontend.
- **Recommendation engine (9.x)** — storage layer ready; runtime
  lives in dddk-frontend.
